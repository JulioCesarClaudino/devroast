# DevRoast Database Setup Guide

This guide will walk you through setting up the Drizzle ORM database implementation for DevRoast.

## Prerequisites

- Docker and Docker Compose installed
- Node.js 18+ installed
- npm or pnpm package manager

## Quick Start (5 Steps)

### 1. Start PostgreSQL with Docker Compose

```bash
docker-compose up -d
```

This will start:
- PostgreSQL 16 on `localhost:5432`
- Adminer (database UI) on `http://localhost:8080`

Verify the connection:
```bash
docker-compose ps
```

### 2. Create `.env.local`

Copy the template:
```bash
cp .env.local.example .env.local
```

The default connection string is already set for local Docker development:
```
DATABASE_URL="postgresql://devroast_user:devroast_password@localhost:5432/devroast"
```

### 3. Install Dependencies

If you haven't already:
```bash
npm install
```

This will install the required packages:
- `drizzle-orm` - ORM library
- `drizzle-kit` - Migration and codegen tools
- `pg` - PostgreSQL driver
- `uuid` - UUID generation

### 4. Generate Initial Migration

Generate the database schema from `src/db/schema.ts`:

```bash
npm run db:generate
```

This creates migration files in `src/db/migrations/` based on your schema.

### 5. Run Migrations

Apply all pending migrations:

```bash
npm run db:migrate
```

This creates all 8 tables in your database.

## Database Structure

The database includes 8 tables:

### Core Tables
- **users** - User accounts
- **languages** - Programming languages
- **roasts** - Code submissions and roasting results

### Related Tables
- **issues** - Code quality issues found in roasts
- **roast_suggestions** - Code improvement suggestions
- **roast_comments** - User comments on roasts
- **roast_favorites** - User likes/favorites

### System Tables
- **migrations** - Auto-generated by Drizzle

For detailed schema information, see `src/db/schema.ts`.

## NPM Scripts

Add these to your `package.json` scripts section:

```json
{
  "scripts": {
    "db:generate": "drizzle-kit generate",
    "db:migrate": "drizzle-kit migrate",
    "db:push": "drizzle-kit push",
    "db:studio": "drizzle-kit studio",
    "db:seed": "tsx src/db/seeds/languages.ts"
  }
}
```

### Available Commands

| Command | Purpose |
|---------|---------|
| `npm run db:generate` | Generate migrations from schema changes |
| `npm run db:migrate` | Run pending migrations |
| `npm run db:push` | Push schema changes directly (development only) |
| `npm run db:studio` | Open Drizzle Studio web UI |
| `npm run db:seed` | Seed database with default languages |

## Using Query Functions

All query functions are server-only ("use server") and can be imported in Server Components or Server Actions.

### Example: Get all languages

```typescript
import { getAllLanguages } from "@/lib/db/language-queries";

export default async function LanguagesList() {
  const languages = await getAllLanguages();
  
  return (
    <ul>
      {languages.map((lang) => (
        <li key={lang.id} style={{ color: lang.color }}>
          {lang.displayName}
        </li>
      ))}
    </ul>
  );
}
```

### Example: Get roast with details

```typescript
import { getRoastById } from "@/lib/db/roast-queries";

export async function RoastDetails({ roastId }: { roastId: string }) {
  const roast = await getRoastById(roastId);
  
  if (!roast) return <div>Roast not found</div>;
  
  return (
    <div>
      <h2>Roast Score: {roast.roastScore}/100</h2>
      <p>Verdict: {roast.verdict}</p>
      <pre>{roast.code}</pre>
    </div>
  );
}
```

## Development Workflow

### After Modifying Schema

1. Make changes to `src/db/schema.ts`
2. Generate migration: `npm run db:generate`
3. Review generated SQL in `src/db/migrations/`
4. Apply changes: `npm run db:migrate`
5. Commit changes:
   ```bash
   git add src/db/schema.ts src/db/migrations/*.sql
   git commit -m "chore(db): update schema with new fields"
   ```

### Creating New Query Functions

1. Create new file in `src/lib/db/` (e.g., `post-queries.ts`)
2. Add `"use server"` directive at the top
3. Import `db` from `@/db/client` and `sql` from `drizzle-orm`
4. Write raw SQL queries with proper parameterization
5. Export async functions that return typed data
6. Use in Server Components or Server Actions

## Troubleshooting

### Database connection fails

**Check if Docker is running:**
```bash
docker-compose ps
```

**View logs:**
```bash
docker-compose logs postgres
```

**Restart containers:**
```bash
docker-compose restart
```

### Migration errors

**Check pending migrations:**
```bash
ls src/db/migrations/
```

**Review migration status:**
```bash
npm run db:studio
```

### Module not found errors

Ensure dependencies are installed:
```bash
npm install
npm install -D drizzle-kit
```

### Port already in use

If port 5432 is already in use:

1. Edit `docker-compose.yml`
2. Change `5432:5432` to `5433:5432` (for example)
3. Update DATABASE_URL in `.env.local`:
   ```
   DATABASE_URL="postgresql://devroast_user:devroast_password@localhost:5433/devroast"
   ```

## Performance Considerations

The schema includes strategic indexes on:
- `username`, `email` (users table)
- `language_id`, `created_at`, `roast_score` (roasts table)
- `roast_id`, `created_at` (comments, favorites, issues)
- `user_id` (comments)

These cover the most common query patterns.
